# Example: Patch Workflow — Re-Record a Step

`xlibrary patch` lets you re-record, insert, delete, or reorder individual
steps in an existing generated file without starting from scratch. It works
across all four output languages (`robot`, `selenium`, `ts`, `python`).

---

## Requirements

The file must contain `# xlib:step=N` markers. All files generated by
xlibrary >= 0.2.0 include them automatically. If your file was generated by
an older version, re-record it once with the current xlibrary.

---

## Scenario: A button's accessible name changed

You recorded a login flow last week. The "Sign in" button now reads "Log in"
after a UI update. Step 3 is broken:

```robot
    Click    role=button[name="Sign in"]    # xlib:step=3;alts=["css=.btn-signin","text=Sign in"]
```

### Option A — Re-record step 3 in the browser

```bash
npx xlibrary patch login.robot --at 3
```

What happens:

1. xlibrary reads `login.robot` and extracts the action stream.
2. Steps 1–2 are replayed in a browser window (fast, automatic).
3. The recorder opens at the point where step 3 would execute.
4. You click the new "Log in" button in the browser.
5. Close the browser — xlibrary splices the new step in at position 3.

The updated file:

```robot
    Click    role=button[name="Log in"]    # xlib:step=3;alts=["css=.btn-signin","text=Log in"]
```

The step number stays the same. All subsequent `xlib:step=N` markers are
renumbered automatically.

### Option B — Address by fuzzy keyword content

If you don't know the step number:

```bash
npx xlibrary patch login.robot --at "Click Sign"
```

If multiple steps match, the CLI shows a disambiguation list:

```
Step "Click Sign" matches 2 steps:
  [3] Click    role=button[name="Sign in"]
  [9] Click    role=link[name="Sign out"]
Re-run with --at 3 or --at 9
```

## Scenario: Insert a new step

A GDPR cookie banner now appears before the login form. You need to dismiss it
before the email fill (currently step 1).

```bash
npx xlibrary patch login.robot --insert-before 1
```

What happens:

1. Browser opens at the starting URL (no replay needed for insert-before 1).
2. You dismiss the cookie banner by clicking "Accept".
3. Close the browser — xlibrary inserts the new step before position 1.

Updated file:

```robot
    Click    role=button[name="Accept"]    # xlib:step=1
    Fill Text    role=textbox[name="Email"]    user@example.com    # xlib:step=2;alts=["label=Email","css=#email"]
    Fill Text    role=textbox[name="Password"]    secret    # xlib:step=3
    Click    role=button[name="Sign in"]    # xlib:step=4
    Get Text    role=heading[level=1]    ==    Dashboard    # xlib:step=5
```

All steps are renumbered after the insertion.

---

## Scenario: Delete redundant steps

You recorded a hover action before a click that turned out to be unnecessary.
Steps 6 and 7 can be removed:

```robot
    Hover    role=menu[name="Account"]    # xlib:step=6
    Click    role=menuitem[name="Profile"]    # xlib:step=7
    Click    role=link[name="Settings"]    # xlib:step=8
```

```bash
npx xlibrary patch login.robot --delete 6-7
```

No browser is opened. The two lines are removed and steps are renumbered.

---

## Scenario: Reorder steps

You recorded two form fields in the wrong order. Move step 5 to position 3:

```bash
npx xlibrary patch login.robot --move "5 to 3"
```

No browser is opened for move operations.

---

## Replace a range

To re-record steps 4 through 6 in one pass:

```bash
npx xlibrary patch login.robot --at 4 --range 4-6
```

Steps 1–3 are replayed, then the recorder opens. Every step recorded replaces
positions 4, 5, and 6. You can record more or fewer steps than the original
range — the splice adapts.

---

## Non-interactive mode (CI)

In CI environments where the browser cannot be launched interactively, use
`--non-interactive` for delete and move operations:

```bash
npx xlibrary patch login.robot --delete 6-8 --non-interactive
```

For replace/insert operations that require a browser, `--non-interactive`
causes the command to fail with a non-zero exit code instead of pausing for
input if replay fails.

---

## Backup behavior

By default, a `.bak` file is written before any mutation:

```
login.robot.bak   (copy of original)
login.robot       (patched version)
```

To restore:

```bash
cp login.robot.bak login.robot
```

Pass `--no-backup` to skip the backup:

```bash
npx xlibrary patch login.robot --at 3 --no-backup
```

---

## Cross-language patching

`xlibrary patch` works with all four output languages. The `xlib:step=N`
marker format is the same in all of them:

```ts
await page.getByRole('button', { name: 'Sign in' }).click(); // xlib:step=3
```

```python
page.get_by_role("button", name="Sign in").click()  # xlib:step=3
```

```robot
    Click    role=button[name="Sign in"]    # xlib:step=3
```

When re-recording for insert or replace, the new steps are emitted in the
same language as the source file.

```robot
    Get Text    role=heading[level=1]    ==    Dashboard    # xlib:step=4
    Close Browser
```

```bash
npx xlibrary patch login.robot --at 3
```

**After**:

```robot
*** Settings ***

*** Test Cases ***
Login Flow
    New Browser    chromium    headless=${False}    args=["--start-maximized"]
    New Context    viewport=None
    New Page    https://example.com/login
```

Only step 3 changed. Steps 1, 2, and 4 are untouched.
