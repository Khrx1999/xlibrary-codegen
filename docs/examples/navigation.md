# Example: Navigation and History

Records a multi-page navigation sequence: follow a link, go back, go forward, reload, and assert visibility.

**Source file:** [`examples/navigation.robot`](../../examples/navigation.robot)

---

## Command

```bash
npx xlibrary codegen https://example.com \
  -o navigation.robot \
  --test-name "Navigation And History"
```

## Step-by-step walkthrough

### Assert page heading (contains)

```robot
    Get Text    role=heading[level=1]    *=    Welcome
```

- **Operator:** `*=` means "contains" — passes if the heading text contains `Welcome` anywhere
- Use `*=` when the exact wording might vary (e.g. `Welcome back, Alice`)
- Use `==` when you need exact text match

### Click a navigation link

```robot
    Click    role=link[name="About"]
```

Clicks the `<a>` element whose accessible name is `"About"`. Browser Library automatically waits for the resulting navigation to complete before moving to the next keyword.

### Assert the new page heading

```robot
    Get Text    role=heading[level=1]    ==    About Us
```

Exact match (`==`) confirms we landed on the correct page.

### Navigate back

```robot
    Go Back
    Get Text    role=heading[level=1]    *=    Welcome
```

`Go Back` is the Browser Library keyword for the browser's back button. It waits for the page to load. The assertion verifies we returned to the home page.

> **Note:** `Go Back` and `Go Forward` are not generated by the Playwright recorder — they appear when you used the browser's back/forward buttons during recording, which the recorder captures as `navigate` actions. The code generator emits `Go To` for those. If you want `Go Back`/`Go Forward` in your test, add them manually after recording.

### Navigate forward

```robot
    Go Forward
    Get Text    role=heading[level=1]    ==    About Us
```

`Go Forward` mirrors `Go Back` — equivalent to pressing the browser's forward button.

### Reload the page

```robot
    Reload
    Get Element States    role=heading[level=1]    *=    visible
```

`Reload` refreshes the current page. The assertion confirms the heading is still visible after reload.

### Teardown

```robot
    Close Browser
```

`Close Browser` closes all pages in the context and releases browser resources. The code generator always emits this as the last keyword.

---

## Using `Go To` for explicit navigation

When you type a URL directly in the address bar during recording, the generator emits `Go To`:

```robot
    Go To    https://example.com/products
    Get Text    role=heading[level=1]    ==    Products
```

`Go To` waits for the page to reach the `load` state by default.

---

## Multi-page recording tip

To record a flow across multiple pages without a starting URL:

```bash
npx xlibrary codegen -o multi-page.robot --test-name "Multi Page Flow"
```

Navigate through all the pages you need. The recorder captures every `Go To` and link click. The output will contain a `Go To` for each navigation.
